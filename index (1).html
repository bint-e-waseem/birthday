Why this exists

The story behind the code

This program didn't start as a portfolio piece. It started as an inside joke that got out of hand: instead of a card, why not a login screen only she could get past? Instead of a paragraph, why not a class called Friendship that never really returns?

Every function in birthday.cpp maps to something real — the six years, the ASCII heart drawn character by character, the "infinite" loop that's secretly capped at five iterations because real programs have to end even when feelings don't, and a memory box that only opens for someone who already knows the password without being told.

"You are the poetry I never knew I could write, the melody that always plays in the background of my life."

That line lives inside a cout statement in the middle of the source file — which is, honestly, the whole point of building it this way.

Under the hood

What the program actually does

🔐 Bestie login screen

Checks a username and password before anything else runs — a small gate that makes the rest feel earned rather than just printed to a terminal.

if (username == "BFF" && password == "MY GIRL") { ... }

💛 The Friendship class

Bundles a name and years-together into an object, then prints the heartfelt opening message and a five-loop "infinite" friendship counter.

class Friendship { string bestieName; int yearsTogether; void showImportance(); };

❤️ Hand-drawn ASCII heart

No image files, no libraries — just nested loops printing spaces and asterisks until a heart shape falls out of the math.

for (int j = 0; j < 2*i+1; j++) cout << "*";

📦 Password-protected memory box

A second, smaller password gate that unlocks a short list of shared memories — a private easter egg inside the bigger one.

class MemoryBox { string accessPassword; bool unlock(); };
Worth knowing: this is a simple learning-project style console program, not a security tool. The username and password are stored as plain text in the source, and a real terminal will show the password on screen as it's typed (there's no input-masking in standard C++ console input). That's a fine tradeoff for a lighthearted birthday surprise between two people — just don't reuse "MY GIRL" as an actual password anywhere that matters, and don't take the pattern into a project that guards something real.

Try it yourself

Compile and run it locally

You'll need a C++ compiler (like g++), available by default on most Linux and Mac systems, or installable via MinGW on Windows.

  1. Download the source file.
    ⬇ birthday.cpp
  2. Compile it.
    g++ -std=c++17 -O2 -o birthday birthday.cpp
  3. Run it.
    ./birthday

    On Windows, run birthday.exe instead.

  4. Log in.

    Username BFF, password MY GIRL — or open the file and swap in your own names and message. It's plain-text C++; that's the whole appeal.
